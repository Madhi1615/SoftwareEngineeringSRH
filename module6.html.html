Module 6: AI-Augmented Software Testing

Time: 3:30 – 4:30 PM | Duration: 45 min teaching + 15 min Q&A

Learning Objectives

  • Understand testing levels (unit, integration, system).
  • Use RPA (UiPath, Blue Prism) for test automation.
  • Generate test cases with LLMs and handle non-deterministic AI outputs.

🔍 Real-World Tools: UiPath & Blue Prism

Robotic Process Automation (RPA) bots mimic human actions in UIs:

  • UiPath: Automates legacy system testing (e.g., SAP, mainframes).
  • Blue Prism: Used in banks for regression testing.

AI Fusion: Combine RPA with LLMs to generate test data: “Create 10 invalid credit card numbers.”

Testing in SWEBOK V4

  • Unit Testing: Test individual functions (e.g., JUnit, pytest).
  • Integration Testing: Test module interactions.
  • System Testing: End-to-end user scenarios.
  • AI Challenge: LLM outputs are non-deterministic → can’t use exact string match.
  • Solution: Semantic similarity checks, snapshot testing, oracles.

Modern Testing Trends

  • LLM-Generated Tests: “Write 5 edge cases for a login function.”
  • Self-Healing Tests: Applitools detects UI changes and auto-updates locators.
  • Test-Driven Development (TDD): Red → Green → Refactor.

🎯 Interactive Activity (10 min)

Generate Test Cases:

  1. Function: `calculateDiscount(age, isStudent)`
  2. In pairs, write 3 test cases (including edge cases).
  3. Then, ask an LLM (if available) or imagine: “What 3 edge cases would an AI suggest?”

❓ Q&A Discussion Prompts (15 min)

  1. Can you have 100% test coverage but still have bugs?
  2. Is manual testing obsolete?
  3. How much testing is “enough”?
  4. Should developers write their own tests?
  5. Does TDD slow down development?
