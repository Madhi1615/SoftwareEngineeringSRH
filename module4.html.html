Core Design Principles (SWEBOK V4)

  • Abstraction: Hide complexity (e.g., car dashboard hides engine details).
  • Modularity: Break system into independent units.
  • Cohesion: Elements in a module belong together (functional cohesion = best).
  • Coupling: Dependencies between modules (data coupling = best, content coupling = worst).
  • KISS, DRY, YAGNI: Keep It Simple, Don‚Äôt Repeat Yourself, You Aren‚Äôt Gonna Need It.

Anti-Patterns

  • God Object: One class does everything.
  • God Team: One team owns everything ‚Üí bottleneck.
  • Spaghetti Code: No structure, impossible to maintain.
