Git Fundamentals (SWEBOK V4)

  • Repository: Single source of truth for code.
  • Branching: Isolate features (e.g., `feature/login`).
  • Merging: Integrate changes; resolve conflicts.
  • Pull Request (PR): Request to merge + code review.
  • Code Review: Catch bugs, share knowledge, enforce standards.

Modern Workflows

  • GitHub Flow: `main` is always deployable; short-lived feature branches.
  • GitLab CI/CD: Auto-run tests on every push.
  • AI in PRs: Copilot suggests PR descriptions; CodeRabbit flags security issues.
